# 【南开大学26C++】器官保卫战 (HealthGame)

## 1. 项目问题描述
随着现代社会生活节奏加快，人们面临诸多健康风险。本项目旨在开发一款基于 C++ 和 Qt 框架的生存策略模拟游戏——《器官保卫战》。玩家需要管理角色的健康指标（血压、血糖、血脂）、免疫力、金钱，以及分配初始天赋（家境、健康、科研、气运），在生老病死的自然规律和突发疾病中尽可能长久地生存，并通过策略博弈（如体检、科研）攻克绝症和寿命极限。

## 2. 项目实际意义
1. **健康教育科普**：通过模拟高压科研对身体指标的负面影响，以及引入“3年一次体检”查出早期癌症的机制，向玩家科普防患于未然、定期体检的现实重要性。
2. **编程能力综合实践**：本项目综合运用了 C++ 面向对象编程、Qt 信号与槽机制、状态机设计模式等核心技术，锻炼了复杂软件架构和工程实践能力。

## 3. 技术路线及算法描述
1. **技术路线**：使用 C++17 标准，基于 Qt Widgets 框架开发纯 2D 图形界面。UI 通过 `QTabWidget` 划分为药剂、手术、保健品等独立模块，实现数据与视图的分离。
2. **核心算法**：
   - **时间驱动引擎**：使用 `QTimer` 作为游戏心跳（Tick），驱动年龄增长、指标衰减和疾病判定。
   - **动态概率模型**：采用 `QRandomGenerator` 结合玩家的“气运”、“健康”天赋，动态修正疾病发生率和受到的伤害值。
   - **隐性疾病状态机**：针对 5 级严重疾病（如癌症），设计了“潜伏期(早期) -> 中期 -> 晚期爆发”的状态机算法。玩家必须主动调用“体检”算法才能在早期干预，否则将在晚期强制爆发，治疗成功率呈指数级下降。

## 4. 开发过程中遇到的难点及解决方法
1. **难点1：多并发健康指标的 UI 联动与精准警报。** 
   - *问题*：初期存在逻辑 Bug，当血压、血糖、血脂中有一项异常时，整个标签页都会标红，无法精准定位。
   - *解决*：引入 HTML 富文本格式动态渲染 `QLabel`，在底层对每项指标进行独立的条件判断和 `<font color='red'>` 着色封装。
2. **难点2：游戏后期的数值膨胀与绝症平衡性设计。** 
   - *问题*：前期金币容易溢出，导致后期治疗毫无生存压力，失去策略性。
   - *解决*：深度重构经济系统。将成年后收入与“科研”天赋绑定，并引入双刃剑机制（科研过高会加速身体损耗）。同时引入多阶段治疗算法，大幅提高高级特效药（如极草冬虫夏草）和绝症手术的费用。
3. **难点3：如何体现 5 级以上疾病的真实感。**
   - *解决*：独创了“隐性疾病进展算法”，疾病在后台静默升级，逼迫玩家养成每 3 年花费 3 万元进行体检的习惯，极大地增强了游戏的现实代入感。
